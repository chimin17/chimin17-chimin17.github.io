Webpack and request module報錯：有關server端與client端Node.js模組

2018-02-22
web
node, webpack

Webpack and request module報錯

一直以來專案開發使用的後端都是.NET Core MVC架構，並用Node.js進行前端的工程
在前端部分整合了webpack, babel及vue等

似乎都沒有發生什麼問題
直到最近遇到一個錯，暸解之後覺得自己有點傻
在此分享一下

主要是看到Node爬蟲的範例，如果用Javascirpt寫爬蟲感覺滿方便的
又有現成code，所以就直接貼

貼在我一個專案的前端的程式區塊
結果，一編譯就發生問題了..

編譯後發生錯誤：

client端與server端模組

在webpack打包時問題顯示說找不到或無法解析fs,net,tls模組
猜想跟引入request有關

但是，fs,net,tls是node.js內建模組啊…怎麼會錯

後來找到這個issue:
https://github.com/webpack/react-starter/issues/3

原來…Node.js有分client端與server端的模組
webpack打包的都是client端，引入request屬於server端應用
原本比較熟悉的後端是.Net架構，開始使用Node幾乎都是用在前端
所以也沒想到什麼前端後端問題，直接把碼貼在js區塊測試一下
報錯…

解決

爬文一下解決方案也是不少
就我這個案例可以使用request替代套件：superagen
https://www.npmjs.com/package/superagent
superagent簡單來講是一個在client端使用的request模組

當然啦，本來就該後端歸後端，前端歸前端XD

後記

會發生這個問題，可能是一直以來不常使用Node的後端模組來開發專案
所以沒有client端與server端模組的想法
過程中還以為Node.js版本太舊之類的

鐵人賽Day 28- WebGL與3D GIS概觀

2018-01-17
web
GIS, WebGL, web, 鐵人賽

3D GIS

WebGIS已經逐步走向3D GIS時代，雖然說3D GIS已經發展很多年了，也有Google Earth這樣的產品，不過真的3D GIS在這幾年才比較有大進展，突破原因除了資料獲取的技術提升以外，主要還是跟Web技術息息相關，其中WebGL佔了重要角色。

3D GIS雖然過去有Google Earth這種霸王級產品，但是其應用有限，而要說這幾年3D GIS發展及前進的速度可說是過去的好幾倍，這與WebGL發展有關，目前3D GIS仍是百家爭鳴階段，連互操作相關議題才剛有幾家大型龍頭爭奪完畢好讓自己的格式成為OGC標準(ESRI I3S勝出)，姑且不論很早起就已ready for use的3D空間分析演算法(與2D差異甚多)，3D GIS在格式、資料方面都與2D截然不同，所以，既然如此，今天就不討論那些，來瀏覽看看一些WebGL for 3D GIS的範例。
