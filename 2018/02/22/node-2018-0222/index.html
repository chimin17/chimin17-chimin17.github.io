關於server side與client side 的Node.js模組

Feb 22, 2018
web
  1. 1. 問題是出在webpack打包時。
  2. 2. client端與server端模組
  3. 3. 解決
  4. 4. 後記

問題是出在webpack打包時。

某天在用Node.js寫一個爬蟲應用時，webpack跳出一個request的錯誤。

錯誤在這邊，編譯發生錯誤：

client端與server端模組

錯誤訊息顯示，fs,net,tls等被request不能被打包，
但這幾個是node.js內建模組啊…
怎麼會錯？

後來找到這個issue:
https://github.com/webpack/react-starter/issues/3

真相大白…
Node.js有分client端與server端的模組
webpack打包的都是client端，引入request屬於server端應用

太少用Node.js寫後端
傻傻的也沒想到什麼前端後端問題，

解決

解決方案也是不少
1.可以找一個request替代套件：superagen
https://www.npmjs.com/package/superagent
superagent簡單來講是一個在client端使用的request模組

2.當然也可以把request寫去後端的部分，再用API接出來給前端用

後記

會發生這個問題，是因為一直以來不常使用Node.js做後端
所以第一時間沒有想到是client端與server端的問題

問題不大，分享一下^^
